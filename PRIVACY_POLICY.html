EveryPaisa — Privacy Policy

Effective date: 24 February 2026

Summary

EveryPaisa is a privacy-first, on-device expense tracker. The app reads bank SMS messages on your device to automatically detect and record transactions. All processing and storage happens locally on your phone — we do not upload SMS content, transaction data, or any personally identifiable information to our servers.

What data we access

  • SMS messages: We request access to SMS only to detect bank transaction messages (permissions: READ_SMS, RECEIVE_SMS).
  • Device authentication: We may use biometric APIs to unlock local features (permission: USE_BIOMETRIC), which uses the Android biometric framework; biometric data never leaves the device.
  • Notifications: Notification permission is used to show app notifications (POST_NOTIFICATIONS).

How we use SMS data

We parse SMS messages to extract transaction fields such as amount, merchant, date/time, and masked account/card identifiers (last 4 digits). Parsing, categorization, analytics, and storage occur entirely on-device. We create a SHA-256 hash of the raw message to detect duplicates; this hash is stored locally and is not transmitted anywhere.

Data sharing

We do not share SMS content, transaction data, or hashed identifiers with third parties. There are no analytics, advertising, or crash-reporting SDKs in the official builds. If you export data (CSV) using the app's export feature, that file is created on your device and can be shared by you manually.

Storage & retention

All data is stored locally in a Room database on your device. You can delete all stored data from the app at any time via Settings > Delete Data. Exported CSV files are stored in your device's shared storage only when you explicitly export; the app does not upload these files.

Security

We use standard Android storage protections. The app generates SHA-256 hashes for deduplication; no encryption is used for network transmission because the app does not transmit data. If you enable biometric protection inside the app, biometric-based unlocking uses Android's secure biometric APIs; no biometric data is stored by EveryPaisa.

Android permissions

  • READ_SMS, RECEIVE_SMS — required to detect bank transaction SMS messages; core functionality.
  • USE_BIOMETRIC — used for optional local authentication.
  • POST_NOTIFICATIONS — used to display notifications on Android 13+.

Why SMS access is necessary

EveryPaisa's primary feature is automatic transaction detection from bank SMS. Alternatives like manual entry or SMS Retriever API are unsuitable because bank transaction SMS vary across banks and often do not use a predictable one-time code flow; the app must read incoming or stored SMS to reliably detect and parse transactions.

Your choices

  • You may decline SMS permissions; the app will still work with manual import or manual transaction entry.
  • You can delete all stored data anytime from Settings.
  • If you export data, you control where that file is saved or shared.
